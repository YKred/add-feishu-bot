# add-feishu-bot Skill - OpenClaw 飞书机器人自动添加工具

一个安全、易用的 OpenClaw 专属 Skill，帮助你一键快速创建并接入新的飞书机器人，**仅支持新增操作，禁止修改/删除已有配置**，全程零手动改配置，保障系统稳定安全。

---

## 📋 功能特性

- ✅ **引导式傻瓜操作**：清晰的使用指南+填写示例，新手零门槛
- ✅ **全模型兼容**：自动读取你系统内所有已配置的大模型，无需手动更新
- ✅ **三重安全校验**：Agent ID、飞书账号ID、工作区目录三重防重复校验，仅新增不修改
- ✅ **自动备份兜底**：每次操作前自动备份主配置文件，改崩可一键回滚
- ✅ **全链路容错**：自动初始化不存在的配置节点，彻底杜绝 undefined 报错
- ✅ **合法ID自动生成**：智能处理中文+数字名称，自动生成合规的机器人唯一ID
- ✅ **流式输出默认开启**：自动为新机器人配置飞书流式输出（打字机效果）
- ✅ **跨Agent调度自动放行**：自动将新机器人加入跨Agent调用白名单，支持多机器人协作

---

## 🔧 前置要求

1. **OpenClaw Gateway**：已完成安装并可正常运行的 OpenClaw 环境
2. **飞书企业账号**：可在飞书开放平台创建企业内部机器人应用
3. **Node.js 运行环境**：OpenClaw 自带的运行时，无需额外安装

## 📦 安装步骤

### 1. 创建 Skill 专属目录
在你的 OpenClaw 根目录下创建技能文件夹：
```bash
mkdir -p /root/.openclaw/skills/add-feishu-bot
```

### 2. 复制技能文件
将以下3个核心文件，完整复制到 `/root/.openclaw/skills/add-feishu-bot/` 目录下：
- `skill.json` - 技能定义与参数配置文件
- `skill.md` - 用户触发时自动展示的引导文档
- `index.js` - 技能核心业务逻辑代码

### 3. 重启网关加载技能
```bash
openclaw gateway restart
```

### 4. 验证安装成功
在飞书中对有权限的主Agent（如虾老大）发送指令：
```
调用 add-feishu-bot 技能
```
若机器人自动返回使用指南与可用模型清单，说明安装成功。

## 🚀 使用方法

### 第一步：触发技能引导
在飞书对话中，对主Agent发送指令，触发技能引导：
```
调用 add-feishu-bot 技能
```
系统会自动返回完整使用指南、参数说明，以及你当前系统内所有可用的模型清单。

### 第二步：按格式提供机器人信息
按照引导模板，完整提供机器人的配置信息，示例如下：
```
我要添加飞书机器人，信息如下：
1. 机器人名字：客服小虾5
2. 飞书App ID：cli_a1234567890abcde
3. 飞书App Secret：1234567890abcdefghijklmnopqrst
4. 模型：ark/doubao-seed-2.0-pro
5. 人设：我是专业的客服机器人，耐心解答用户所有问题
6. 表情：🤝
```

### 第三步：等待自动完成全流程配置
技能收到信息后，会全自动执行以下操作，无需任何手动干预：
1.  全量安全校验（ID、目录是否已存在）
2.  自动备份原 `openclaw.json` 配置文件
3.  创建机器人专属独立工作区目录
4.  新增 Agent 配置（人设、模型、工作区）
5.  新增飞书账号配置（凭证、流式输出）
6.  新增消息路由绑定规则
7.  更新跨Agent调用白名单
8.  保存新配置并重启 OpenClaw 网关生效
9.  返回最终创建结果与配置信息

## ⚙️ 参数详细说明

| 参数名 | 是否必填 | 参数说明 | 填写示例 |
|--------|----------|----------|----------|
| `botName` | ✅ 必填 | 机器人显示名称，支持中文、数字 | 客服小虾5、技术支持虾、长文档解读助手 |
| `appId` | ✅ 必填 | 飞书开放平台获取的 `cli_` 开头的App ID | cli_a1234567890abcde |
| `appSecret` | ✅ 必填 | 与App ID配套的飞书App Secret密钥 | 1234567890abcdefghijklmnopqrst |
| `model` | ❌ 选填 | 机器人使用的大模型完整ID，不填则使用系统默认模型 | ark/doubao-seed-2.0-pro、volcengine-plan/kimi-k2.5 |
| `theme` | ❌ 选填 | 机器人人设/主题描述，不填使用默认通用人设 | 我是专业的代码助手，帮你写代码、改bug、解释脚本 |
| `emoji` | ❌ 选填 | 机器人头像表情，不填默认使用 🤖 | 📝、💻、🔧、🤝 |

## 🔒 安全机制

### 三重前置安全检查
所有修改操作执行前，会先完成以下校验，任意一项不通过直接终止操作，绝不修改现有配置：
1.  **Agent ID 唯一性检查**：校验生成的机器人唯一ID是否已被占用
2.  **飞书账号ID 唯一性检查**：校验飞书账号ID是否已存在于配置中
3.  **工作区目录 唯一性检查**：校验机器人专属工作区目录是否已存在

### 仅新增、不修改、不删除原则
- 所有配置操作仅执行数组 `push` 新增、对象新键新增操作
- 绝不修改、覆盖、删除配置文件中已有的任何配置项
- 每次操作前自动生成带时间戳的配置备份，兜底防翻车

## 🔧 自定义配置说明

### 路径自定义
若你的 OpenClaw 路径与默认路径不同，可修改 `index.js` 开头的路径配置：
```javascript
// 主配置文件路径
const CONFIG_PATH = '/root/.openclaw/openclaw.json';
// 机器人工作区根目录
const WORKSPACE_BASE = '/root/.openclaw/workspace/workspace';
```

### 中文拼音映射扩展
若需要支持更多中文名称的ID生成，可扩展 `index.js` 中的 `PINYIN_MAP` 拼音映射表：
```javascript
const PINYIN_MAP = {
  '助':'zhu','理':'li','客':'ke','服':'fu',
  '技':'ji','术':'shu','运':'yun','营':'ying',
  // 在此处添加更多中文-拼音映射
};
```

## ❓ 常见问题排查

### Q: 调用技能提示「模型不存在」怎么办？
A: 请确保填写的模型完整ID（格式：`提供商名称/模型ID`），与 `openclaw.json` 中 `models.providers` 里配置的完全一致，大小写、连字符不能有偏差；同时确认该模型已完成提供商配置与权限开通。

### Q: 技能返回「安全拦截」提示怎么办？
A: 说明你填写的机器人名称，生成的ID、工作区目录或飞书账号已存在于系统中，更换一个全新的机器人名称重试即可。

### Q: 配置改崩了，怎么回滚？
A: 技能每次操作前都会自动生成备份文件，在 `/root/.openclaw/` 目录下找到最新的 `openclaw.json.backup.时间戳` 文件，替换当前的 `openclaw.json`，重启网关即可恢复。

### Q: 机器人创建成功，但飞书不回复消息怎么办？
A: 请按以下步骤排查：
1.  飞书开放平台已开通机器人相关权限，并完成企业管理员审批
2.  已订阅「接收消息v2.0」事件，且配置了长连接接收模式
3.  应用已完成版本发布，可用范围包含对话的用户/群聊
4.  查看 OpenClaw 网关日志，确认无模型调用、权限相关报错
